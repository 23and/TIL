###파일시스템
- 플로피디스크, 하드디스크 등의 대용량 보조기억장치들이 등장하면서, 프로그램의 내용을 전원이 차단된 이후 시점에서도 저장,
불러오기가 가능해졌다. 
- 파일시스템으로 데이터를 관리할경우 데이터의 중첩도 많아지고 , 동시접근과 공유가 어려워지는 단점이 있다. 

###DB
- 여러 사람에 의해 공유되어 사용될 목적으로 통합 및 관리되는 데이터 집합
- 조직의 여러 응용 시스템이 공용(Shared)하기 위해 최소의 중복으로 통합(Integrated), 저장(Stored), 운영(Operation)되는 데이터 집합

###파일시스템과 데이터베이스의 비교
- 파일 시스템
 - 장점 : 
  - 처리속도가 DB에 비해 빠르다.
  - 구현이 간편하다
  - 비용이 저렴하다.
 
 - 단점 : 
  - 자료의 중복 발생
  - 데이터 무결성 보장의 어려움
  - 개발 생산성을 기대하기 어려움

- 데이터베이스
 - 장점 
  - 자료의 독립성
  - 데이터 무결성 보장
  - 개발 생산성 보장 (표준 SQL, 개발기간 절감, 운영비용 절감)
 - 단점 
  - 시스템의 부하 및 복잡성
  - 별도의 관리 이력 필요
  - 추가 도입비용 발생

###DAS
- 스토리지 공유 가능
- 소규모의 독립적인 구성에 적합

###NAS
- 스토리지 공유 가능
- 파일공유를 위한 가장 안정적, 신뢰성 높음
- LAN을 통해 Storage를접속한다는 개념

###SAN
- 스토리지 공유 가능
- 구성의 유연성, 확장성, 관리 편리 (왜..?)
- DAS의 접속 한계를 극복하여Storage접속 서버수 증가, 서버당 접속 Storage의 증가와 아울러 관리 비용의 절감을 목적으로 채널 접속 인터페이스 중에서 FiberChannel접속에 네트워크의 개념을 도입함

###파이프라인
컴퓨터에서, 파이프라인이란 프로세서로 가는 명령어들의 움직임, 또는 명령어를 수행하기 위해 프로세서에 의해 취해진 산술적인 단계가 연속적이고, 다소 겹치는 것을 말한다. 파이프라인이 없다면 컴퓨터의 프로세서는 메모리에서 첫 번째 명령어를 가지고 와서, 그것이 요구하는 연산을 수행하고, 그리고 나서 다음번 명령어를 메모리로부터 가져오는 식으로 동작한다. 명령어를 가져오는 동안에, 프로세서의 산술연산부분은 다음 명령어가 도착되기를 기다리며 쉬어야만 한다. 파이프라인을 쓰면, 컴퓨터 구조는 프로세서가 산술연산을 수행하는 동안에 다음번 명령어를 가져올 수 있으며, 그것을 다음 명령어 연산이 수행될 수 있을 때까지 프로세서 근처의 버퍼에 가져다놓는다. 명령어를 가져오는 단계는 끊임없이 계속된다. 그 결과, 주어진 시간동안에 수행될 수 있는 명령어의 수가 증가한다.
