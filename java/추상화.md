#추상화
- 클래스의 공통부분을 뽑아내서 조상클래스를 만드는 것
- 실세계에 존재하는 객체들을 프로그램으로 변환하기 위해서 객체들의 공통된 특성을 파악하고 불필요한 특성을 제거하는 과정을 거친다. 이러한 과정을 추상화라고한다. 이 과정을 통해서 객체들의 공통된 특성들을 모아놓은 것을 클래스라고한다

###추상클래스
- 미완성 설계도
- 완성되지 못한채로 남겨진 설계도
- 미완성 메소드를 포함
- 추상클래스는 인스턴스를 생성할 수 없고 상속통해서 완성시킬 수 있다
````java
abstract class 클래스이름{ }
````
- 추상클래스로부터 상속받는 자손클래스는 오버라이딩을 통해 조상인 추상클래스의 추상메소드를 모두 구현해주어야한다.
- 추상클래스에는 일반메소드, 인스턴스변수도 정의할 수 있다
- 추상클래스에는 일반메소드도 정의할 수 있다
- Shape 추상클래스 , Circle,Rect는 추상클래스를 구현한 클래스일 때, Circle과 Rect클래스에 공통적으로 move()라는 클래스를 넣고싶다면
 - Shape 클래스에 move()메소드를 정의해준다.


###추상클래스는 왜 사용하는 것인가?
- 메소드를 작성할 때 선언부를 결정하는 것은 중요하다
- 메소드의 이름과 메소드의 작업에 필요한 매개변수 작업결과로 어떤 타입의 값을 반환할 것인지 결정하는 것은 쉽지않다.
- 메소드를 사용하는 쪽에서 메소드가 실제로 어떻게 구현되어있는지 몰라도 메소드의 이름과 매개변수 리턴타입 즉 선언부만 알고 있으면 되므로
- 내용없는 추상메소드를 사용하는 코드를 작성하는 것이 가능하다.
- 상속을 통해서 프로그램의 개발 유지 보수를 쉽게 해준다

###인터페이스를 사용하는 이유
- 다중상속을 가능하게한다
 - 어떤 가게는 물건을 팔 때 돈 뿐만 아니라 모든 경제적 가치가 있는 것을 받는다
 - 이것을 함수로 작성하면 shop클래스의 sell()메소드는 매개변수로 Money 타입뿐만 아니라 다른 자료형 (예 watch)을 받을 수 있어야한다
 - 이때 Money와 Watch는 모두 물건이라는 Thing클래스로부터 상속을 받는다
 - 따라서 sell(Thing v)형태로 작성한다면 돈뿐만 아니라 시계도 받을 수 있다.
 - 하지만 쓰레기(trash)클래스도 Thing으로부터 상속받는다 따라서 shell(Thing v)메소드는 쓰레기도 매개변수로 받을 수 있는 문제가 발생한다. 
 - 이러한 문제는 다중상속을 사용하면 간단하게 해결할 수 있다 예를 들어 Money와 Watch는 Thing으로부터 상속을 받으면서 다른 것 (예 Valuable)으로부터 상속받는다면 메소드를 sell(Valuable v)형태로 작성할 수 있을 것이다. 
 - 그런데 자바에서는 클래스의 단일 상속만 가능하기 때문에 다중상속을 지원하기 위해서 interface를 사용하면 된다

###클래스란 무엇인가? 인터페이스란 무엇인가?
- 이론적인 측면에서 객체는 코드를 갖는 구체적인 객체와 실제 코드를 갖지않는 추상적인 객체로 구분된다
- 이 두가지 형태의 객체는 모두 데이터타입을 정의하기위해서 사용될 수 있다. 
- 구체적인 객체는 자바에서 클래스로 표현되고 추상적인 객체는 자바에서 인터페이스로 표현된다 
- 클래스에서 상속과 마찬가지로 인터페이스의 구현은 새로운 서브타입을 정의하기 위해서 사용된다

###추상클래스 인터페이스 비교
- 인터페이스 
 - 디자인 
 - 다중 상속 가능
- 추상클래스 
 - 일부 구현 일부 디자인
 - 단일상속

###인터페이스의 특성
- 인터페이스의 멤버필드는 디폴트로 staitc final으로 선언되어 있다. 따라서 인터페이스 멤버필드값을 변경하려는 시도는 컴파일 에러를 발생시킨다
- 인터페이스의 구현도 상속과 같은 개념이기 때문에 인터페이스를 구현한 서브클래스는 인터페이스의 멤버필드를 상속받아서 사용할 수 있다
- 서브클래스가 슈퍼클래스로부터 상속받고 인터페이스를 구현하는 경우에 동일한 이름의 멤버필드가 슈퍼클래스에도 선언되어있고 인터페이스에도 선언되어있으면, 서브클래스에서 이 변수를 사용하는 경우에 어떤 변수인지 명확히 지정하지않을 경우 컴파일 에러가 발생한다. 이렇게 동일한 이름의 변수가 사용되는 경우에는 변수가 어떤 것인가를 정확히 기술해주어야한다. 슈퍼클래스의 변수를 지칭할 때는 super키워드를 사용하고 인터페이스의 변수를 지칭할 때는 interfacename.member_filed형태로 사용한다
- 여러개의 인터페이스는 필요한 경우 콤마를 이용해서 여러개를 기술 할 수 있다.
- 인터페이스사이에도 상속관계가 있을 수 있다. 인터페이스에서 상속은 클래스와 마찬가지로 extends를 사용한다
