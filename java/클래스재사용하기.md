#[java] 클래스 재사용하기

- 혁신적인 재사용이 되려면 단순히 코드를 복사하여 수정하는 것보다 많은 일을 할 수 있어야 한다.

- C(절차적인 언어) : 코드 복사와 수정에 의한 재사용

- 자바 : 클래스 재사용

- 기존 코드를 건드리지 않고 클래스를 재사용하는 방법

 - 컴포지션(composition) : 새 클래스의 내부에 기존 클래스의 객체를 생성한다. 새클래스가 기존 클래스의 객체로 구성되기 때문이다. 형식이나 개념이 아닌 코드의 기능을 재사용 하는 것이다.
 - 상속(inheritance) : 새 클래스를 기존 클래스의 한 타입으로 만든다. 기존 클래스의 형태를 가져온 후 그것을 수정하지 않고 거기에 코드를 추가한다. 

- 객체 참조의 초기화를 위해 컴파일러가 디폴트로 객체 생성을 하지 않는 것은 현명한 일이다. 많은 경우 불필요한 부담을 초래하기 때문이다.
- 객체참조 초기화를 해야한 다면
 - 객체 참조가 정의되는 시점에 한다. 생성자가 호출되기 전에 객체가 항상 초기화 될것이라는 의미이다.
 -  클래스의 생성자에서 초기화 한다.
 -  객체를 실제로 사용하기 바로 직전에 초기화 한다. 이를 늦 초기화라고 하며 객체 생성에 시간이 많이 소요되거나, 객체가 매번 생성될 필요가 없는 경우 부담을 줄일 수 있다.
 - 인스턴스 초기화를 사용한다.

- 하나의 프로그램에 여러개의 클래스가 있더라도 명령행에서 준 클래스의 main()메소드가 호출된다.

- 상속을 허용하려면 일반적으로 베이스클래스의 모든 필드는 private으로 하고, 메소드는 public으로 한다.

- 자식 클래스에서 오버라이딩 한 메소드를 호출할 때, 앞에 super 키워드를 사용하지 않으면, 재귀 호출된다. 부모 클래스의 메소드를 호출하고 싶으면 super 키워드를 사용해야한다.

- 자바 컴파일러는 파생 클래스의 생성자 내부에 베이스 클래스의 생성자 호출을 자동으로 추가한다. 베이스클래스 부터 객체가 생성된다.
