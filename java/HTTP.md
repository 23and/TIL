#HTTP
- HTTP는 인터넷을 통해 데이터를 요청하고 받기 위한 주된 방법 중 하나이다.
- HTTP는 월드와이드웹을 통해 소통하는데 사용된다. 아파치의 HTTPD, 마이크로소프트의 IIS, 톰캣 같은 HTTP 서버는 체계적으로 들어오는 요청을 수신해서 적절한 응답을 할 수 있도록 구성되어 있다.

###HTTP 메서드
```
GET : 요청된 자원을 제공한다. 변경이 없어야 한다.
POST : 요청된 콘텐츠를 위한 자원을 업데이트한다.
PUT : 요청된 콘텐츠를 위한 자원의 내용을 새로 설정한다.
DELETE : 요청된 자원을 삭제한다.
HEAD : GET 요청과 비슷하지만 응답 코드와 헤더만 반환한다.
```
###HEAD는 언제 쓸까
- 다운로드 필요없이 대용량 자원이 존재하는지 확인하는데 사용할 수 있다. 
- 클라이언트가 이전에 해당자원을 다운로드 했다면 업데이트된 최신 사본이 있는지를 클라이언트가 알 수 있게 **Last-Modified** 헤더를 확인할 수 있다.

###HTTP응답코드
- HTTP 응답의 첫번째 행은 응답 코드를 포함하며 코드가 의미하는 내용이 무엇인지 설명한다.
- 첫번째 숫자는 응답분류, 모든 2xx 코드는 요청과 응답이 성공적이라는 것을 알려준다.
- 4xx분류는 클라이언트에게 에러, 유요한 요청을 만들지 못했다는 것을 알려준다.
- 5xx분류는 서버 측에 이슈가 있다는 것을 알려준다. 이경우 서버에 조치를 취하지 않고서는 클라이언트가 예외상황을 해결할 수 없다.
